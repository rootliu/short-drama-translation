# 短剧剧本适配系统：从网文到爆款短剧的"评书艺人"AI

## 研究叙事 (Research Narrative)

---

### 问题的起源

中国网文产业年产值超过300亿元，数百万网文作者创造了海量故事。与此同时，短剧市场在2025年已达50亿美元规模，预计2033年将增长至376亿美元。ReelShort单平台MAU达5000万，年收入约4亿美元。

一个显而易见的机会出现了：**网文作者希望其作品被改编为短剧以获得增收**。但网文和短剧是完全不同的叙事形式——就像杨家将的传说故事与评书《杨家将》是不同的艺术形式。

评书艺人拿到杨家将的故事梗概后，并不是照本宣科。他们做了几件关键的事：

1. **结构重组** — 将长篇故事拆分为适合"长篇连播"的集数结构
2. **情感增补** — 在原故事的骨架上添加情感高潮，如穆桂英挂帅、大破天门阵等原著中没有的内容
3. **钩子设计** — 每回结尾"且听下回分解"，制造悬念让听众明天继续来
4. **受众适配** — 根据地域、听众群体特点调整故事的引入方式和重点
5. **填充空白** — 原故事中情感不够饱满的地方，用合理的桥段补充

**我们要做的，就是一个AI评书艺人。**

输入不是空白画布，而是一部已有的网文小说。系统需要像经验丰富的评书艺人一样，将这个故事"改编"为能够产生高留存率和高转化率的短剧剧本。

### 核心假设

**假设一：短剧的成功可以被量化建模**

50集以上的短剧存在可识别的模式来吸引并留住观众。除了剧情基础和演员，其风格和情感脉络发展决定了观众认可程度。这个认可程度由两个指标定义：
- **留存率** — 观众持续观看的比例曲线
- **转化率** — 被导流来的观众持续观看50集以上的比例

**假设二：情感管理是核心机制**

吸引用户的核心是共情。情感脉络管理包含三个层面：

| 层面 | 机制 | 量化目标 |
|------|------|---------|
| 情感攀升 | 主要情感从低到高的发展 | 攀升速率和峰值 |
| 高位维持 | 在高位保持并平稳转化 | 高位持续时长 |
| 情感反转 | 从一个情感高位突然转到另一个情感高位 | **爽度值** |

**爽度值的定义**：当观众跟随角色经历某种高位情感（如压抑、委屈），突然通过一个trigger事件转换到另一种高位情感（如自豪、解气）时产生的情感冲击力。

例：某小哥一直被欺负（压抑高位），他下意识救起了一个小动物（trigger），有人因此发现他是隐忍的武林高手（反转），观众的情感从压抑高位瞬间转换为自豪/爽感高位。这个delta值就是一次"爽度"。

50集以上的成功短剧需要让这个爽度值一直处于高位——不是一次性释放，而是像过山车一样持续波动。

**假设三：Hook是留存的直接驱动**

Hook有两个功能：
1. **维持功能** — 进行情感转换，保持爽度值
2. **引出功能** — 在某个情感从高位下降时，补充观众的"失落值"，或保持高位因素，引出继续观看下一集的兴趣

### 文献基础

#### 情感弧线：已有量化方法

Reagan et al. (2016) 分析了1,327部小说，识别出6种基本情感弧线形状。更重要的是，**特定弧线形状与更高的下载量（受欢迎度）显著相关**。Zimmerman (2026) 将样本扩大到28,606本书，识别出8种叙事原型，并发现一个关键结论：**叙事轨迹的方差（情感波动幅度）是读者数量的最强预测因子**（partial rho=0.32）。

这直接支持"爽度值"假设——不是哪种情感最好，而是**情感波动越剧烈，越吸引人**。

#### 悬念与Hook：信息论框架

Wilmot & Keller (2020) 证明悬念最好建模为**前瞻性不确定性缩减**——观众对"接下来会怎样"的期待，而非对"刚发生了什么"的惊讶。Schulz et al. (NeurIPS 2024) 进一步用信息论框架**形式化度量了plot twist、cliffhanger和关键转折点**，在电视剧数据上得到验证。

这提供了Hook评分的数学基础。

#### 情感轨迹可以作为生成约束

Brahman & Chaturvedi (2020) 首次证明神经网络模型可以**遵循预设的情感弧线同时保持叙事质量**。Wen et al. (2025) 在游戏叙事生成中用"Rise and Fall"情感模式指导分支故事发展，参与者评估显示**情感弧线整合显著提升了参与度、叙事连贯性和情感影响力**。

#### LLM的已知局限——也是我们的设计约束

- LLM叙事**缺乏张力和多样性**（EMNLP 2024）
- LLM生成中文文学出现**结构同质化**（Ma et al. 2026）
- GPT-4在大规模叙事中**无法整合角色动机与戏剧张力**（IEEE CoG 2025）

但关键的是——我们不需要LLM从零创作。原始故事已经存在（网文）。LLM的角色是"评书艺人"：在已有故事骨架上做结构化增补，这比from-scratch创作容易得多。

#### 短剧行业的实践智慧

中国短剧行业已经沉淀出经验法则：
- **黄金3秒定律**：开头3秒建立冲突
- **15秒反转规则**：15秒内必须有反转
- **付费卡点**：通常在第8-12集（首个重大反转后）
- **制作周期**：7-15天从剧本到上线
- **爆款要素**：复仇动机、地位反转、禁忌恋情、身份揭露、权力幻想

### 研究目标

#### 目标一：建立短剧成功模式的量化Benchmark

**Shuang Benchmark** — 短剧"爽度"量化评估框架

输入：一部短剧的完整剧本/字幕
输出：
- 情感曲线图（逐集+逐场景的多维度情感强度）
- 爽度值时序（每次情感反转的delta值及其分布）
- Hook评分（每集结尾的悬念信息量）
- 模式匹配度（与已知成功模式的拟合度）
- 预测评分（基于量化特征的预期留存率/转化率区间）

验证方法：用已知的高转化率短剧和低转化率短剧进行对比验证，检验Benchmark的区分能力。

#### 目标二：建立网文→短剧的改编框架

**评书艺人框架** — 从网文到短剧的结构化改编方法

这不是单一框架，而是可能的多个模式模板（对应不同类型/受众）：

| 模式 | 适用类型 | 核心节奏 | 爽度分布 |
|------|---------|---------|---------|
| 复仇阶梯 | 逆袭/重生/复仇 | 每3集一个"欺负→逆袭"循环，逐级升级 | 均匀高频 |
| 身份剥洋葱 | 赘婿/隐藏身份 | 渐进式身份揭露，每次揭露产生更大反转 | 递增式 |
| 虐恋过山车 | 霸总/甜宠 | 甜→虐→甜循环，虐的深度逐渐加深 | 高幅振荡 |
| 升级打怪 | 修仙/武侠/格斗 | 固定的"修炼→挑战→突破"节奏 | 脉冲式 |
| 悬疑递进 | 穿越/悬疑 | 信息逐步释放，每次释放打开新谜题 | 阶梯式 |

每个模式定义：
- 集级别的情感目标序列
- 反转点的推荐位置和类型
- Hook的推荐策略
- 付费卡点的最优位置

#### 目标三：构建"AI评书艺人"原型Agent

**输入**：
- 网文原文（或梗概/大纲）
- 目标模式（从目标二的模板中选择）
- 目标受众画像（类型偏好、年龄、地域）

**处理过程**（像评书艺人一样）：

```
第一步：骨架提取
  将网文拆解为核心情节点、角色关系、关键冲突
  识别原故事的内在情感弧线

第二步：模式映射
  将原故事映射到目标短剧模式模板
  识别哪些情节点可以直接对应模板的关键位置
  标记"空白区域"——模板中需要但原故事中没有的情感内容

第三步：增补填充（"穆桂英挂帅"式增补）
  对空白区域进行内容增补：
  - 参考原故事的世界观和角色设定
  - 参考互联网上同类型作品的流行桥段
  - 确保增补内容与原故事逻辑一致
  增补的目的：填满短剧情感管理的需求

第四步：Hook工程
  将每集结尾改造为钩子形式：
  - 重点反转设计为集内高潮
  - 剧情结尾设计为悬念钩子
  - 确保爽度值在50集跨度上持续高位

第五步：Benchmark自检
  用目标一的Benchmark框架对生成的剧本评分
  如果不达标，反馈调整第二~四步
```

**输出**：
- 完整的短剧剧本（50-100集，每集1-2分钟）
- Benchmark评分报告
- 每集的情感曲线和Hook标注
- 与原网文的映射关系图（标记哪些是原内容、哪些是增补）
- 不同受众群体的预期转化率区间

### 评书艺人的隐喻：为什么这个类比是精确的

评书艺人和我们的系统有以下精确对应：

| 评书艺人 | AI评书艺人系统 |
|---------|---------------|
| 拿到杨家将的故事梗概 | 输入网文原文/梗概 |
| 根据听众群（南方/北方/书场/广播） | 根据目标受众画像 |
| 将故事拆成"回" | 将故事拆成集 |
| 添加穆桂英挂帅等增补情节 | AI增补填充空白区域 |
| "且听下回分解" | Hook工程 |
| 在关键处"拍惊堂木" | 情感爆破点设计 |
| 根据现场反馈调整节奏 | 根据受众画像调整模式 |
| 不同艺人同一故事不同版本 | 同一网文多模式输出 |

评书的核心智慧是：**故事的骨架是给定的，但骨架上的肉——节奏、情感、悬念、增补——决定了是否能留住听众。**

短剧同理。网文提供了骨架，AI评书艺人的工作是把这个骨架变成能留住观众、产生付费转化的短剧。

### 技术路径

#### Phase 1: Benchmark框架 — "度量衡"

**目标**：能够量化评价任意短剧剧本的"爆款潜力"

```
输入: 短剧字幕/剧本文本
        │
        ▼
┌─────────────────────────┐
│  S1: 情感标注            │  逐台词标注情感维度和强度
│  (LLM逐句分析)          │  输出: emotion_timeline.json
└────────────┬────────────┘
             ▼
┌─────────────────────────┐
│  S2: 弧线拟合            │  将情感时序拟合到8种标准弧线
│  (统计分析)              │  输出: arc_profile.json
└────────────┬────────────┘
             ▼
┌─────────────────────────┐
│  S3: 爽度值计算          │  识别情感反转点，计算delta
│  (信息论方法)            │  输出: shuang_timeline.json
└────────────┬────────────┘
             ▼
┌─────────────────────────┐
│  S4: Hook评分            │  每集结尾的悬念信息量
│  (不确定性缩减模型)       │  输出: hook_scores.json
└────────────┬────────────┘
             ▼
┌─────────────────────────┐
│  S5: 综合评分            │  加权汇总 + 模式匹配
│                         │  输出: benchmark_report.json
└─────────────────────────┘
```

**需要的资源**：
- 5-10部短剧字幕（高转化 + 低转化各半，用于验证框架区分度）
- 情感维度定义（建议8维：愤怒、悲伤、快乐、恐惧、惊讶、厌恶、压抑/委屈、爽/解气）
- Gemini API（已有）

#### Phase 2: 改编框架 — "模式库"

**目标**：建立从网文到短剧的映射模板

```
输入: 网文全文 + 目标模式
        │
        ▼
┌─────────────────────────┐
│  A1: 骨架提取            │  提取情节点、角色、冲突
│  (LLM摘要 + NER)        │  输出: skeleton.json
└────────────┬────────────┘
             ▼
┌─────────────────────────┐
│  A2: 模式映射            │  将情节点对齐到模板时间轴
│  (约束优化)              │  输出: mapping.json
└────────────┬────────────┘
             ▼
┌─────────────────────────┐
│  A3: 空白识别            │  标记需要增补的位置和类型
│  (差异分析)              │  输出: gaps.json
└─────────────────────────┘
```

**需要的资源**：
- 2-3部已成功改编为短剧的网文（对照原文和短剧版本，提取改编规律）
- 各类型短剧的模式模板（从Phase 1的Benchmark分析中提炼）

#### Phase 3: AI评书艺人Agent — "生成器"

**目标**：将网文自动改编为短剧剧本

```
┌──────────────────────────────────────────────────────┐
│                  Orchestrator                         │
│  管理整个改编流程，协调各子Agent                        │
└──────┬───────────┬───────────┬───────────┬───────────┘
       ▼           ▼           ▼           ▼
┌────────────┐┌────────────┐┌────────────┐┌────────────┐
│ Skeleton   ││ Augmentor  ││ Hook       ││ Evaluator  │
│ Agent      ││ Agent      ││ Engineer   ││ Agent      │
│            ││            ││            ││            │
│ 提取网文    ││ 填充空白    ││ 设计每集    ││ Benchmark  │
│ 骨架并映    ││ 增补情节    ││ 结尾钩子    ││ 评分并反馈  │
│ 射到模板    ││ 补充情感    ││ 设计反转    ││ 是否达标    │
└────────────┘└────────────┘└────────────┘└────────────┘
       │           │           │                │
       │     参考互联网同类型    │                │
       │     作品的流行桥段     │                │
       └───────────┴───────────┴────────────────┘
                          │
                          ▼
                   短剧剧本 + 评分报告
```

**Augmentor Agent（增补Agent）的特殊设计**：

这是整个系统最核心的部件，对应评书艺人"添加穆桂英挂帅"的能力。它需要：

1. **理解原故事的世界观边界** — 增补的内容不能违背原设定
2. **参考同类型流行桥段** — 从互联网上的同类网文/短剧中提取受欢迎的情节模式
3. **服务于情感目标** — 增补不是随意的，是为了填满模板中的情感空白
4. **保持因果链** — 增补的trigger事件必须在前文有伏笔或合理性

### 与现有系统的关系

本研究构建在已有的短剧翻译Pipeline之上。现有系统已经实现了：
- 7阶段DAG流水线（字幕→角色→情感→翻译→情感分析→Hook→QA）
- Gemini AI集成
- 文件驱动的存储架构
- SSE实时进度反馈

Benchmark框架可以直接复用现有的情感分析（S3）和Hook分析（S7）阶段，将其从"分析单集"扩展为"分析整部剧的情感时序"。改编Agent则是在此基础上的新增能力。

---

### 预期成果

1. **Shuang Benchmark** — 首个针对短剧的情感量化评估框架
   - 可量化评价任意短剧剧本的爆款潜力
   - 可区分高转化和低转化短剧（验证指标：AUC > 0.8）

2. **改编模式库** — 5种以上从网文到短剧的标准改编模式
   - 每种模式包含情感目标序列、反转点模板、Hook策略

3. **AI评书艺人原型** — 可运行的改编系统
   - 输入网文 → 输出短剧剧本 + Benchmark评分
   - 支持多模式输出（同一网文 → 不同风格的短剧版本）
   - 标注原内容 vs 增补内容的映射关系

4. **研究论文/技术报告** — 系统化的方法论文档
   - 情感弧线量化方法
   - 爽度值的数学定义
   - Benchmark框架的有效性验证
