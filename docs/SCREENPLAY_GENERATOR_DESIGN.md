### 1.1 与现有工作的关系


### 1.2 产品目标

**输入**：网文小说（或大纲/梗概）
**输出**：可直接交给导演和制片团队的分镜剧本包，包含：

1. **分集分镜剧本**（每集1-3分钟，标准化格式）
2. **角色特质卡**（便于道具/服装/演员理解角色）
3. **评估报告**（爽度曲线、Hook强度、角色一致性等）
4. **Mock字幕**（用于配音/字幕团队参考）

### 1.3 为什么需要分镜级剧本

传统短剧剧本只提供对白和简单场景描述，拍摄现场需要导演自行设计镜头和演员动线。我们的系统要产出**分镜级**剧本，意味着：

| 传统剧本 | 分镜剧本（本系统） |
|---------|-----------------|
| "苏晚晴走进办公室" | "苏晚晴从画面左侧入画，缓步走向办公桌，镜头从全景推至中景，在她坐下时切到面部特写" |
| 角色说了什么 | 角色怎么说的（语气、停顿、潜台词标注） |
| 一个场景描述 | 场景的光线/色调/道具/氛围的详细设定 |
| 无镜头设计 | 每个动作对应的具体镜头（景别、运镜、角度） |

**价值**：减少拍摄现场的决策负担，让低成本团队也能拍出高质量短剧。

### 2.2 本设计新增的理论补充

#### 2.2.1 影视分镜理论

分镜（Storyboard）源自迪士尼动画制作，后成为电影工业标准。核心要素：

- **Shot Size（景别）**：远景/全景/中景/近景/特写/大特写
- **Camera Movement（运镜）**：推/拉/摇/移/跟/升降/环绕/手持
- **Camera Angle（角度）**：平视/俯拍/仰拍/倾斜/主观镜头
- **Blocking（动线/走位）**：演员在场景中的移动路径和位置关系
- **Mise-en-scène（场面调度）**：场景内所有视觉元素的安排

**短剧特有约束**：
- 竖屏格式（9:16），构图法则与横屏完全不同
- 每集1-3分钟，信息密度极高，平均3-5个镜头变化/分钟
- 手机观看为主，特写和近景占比远高于传统影视
- 前3秒决定是否继续观看（"黄金3秒法则"）

#### 2.2.2 短剧编剧方法论

来自行业实践的核心原则：

| 原则 | 说明 | 系统实现 |
|------|------|---------|
| 黄金3秒 | 开头3秒必须建立冲突/悬念 | 每集第一个镜头设计为冲突引入 |
| 15秒反转 | 15秒内必须有情感变化 | 第一个Pivot点约束在15秒内 |
| 付费卡点 | 第8-12集首个重大反转 | EP8-10的爽度值必须达到峰值 |
| 集尾Hook | 每集结尾必须有未完成感 | Suspense度量约束 |
| 角色标签化 | 每个角色有鲜明记忆点 | 角色特质卡（外形/口头禅/标志性动作） |
| 视觉信息优先 | 能用画面表达的不用对白 | 旁白仅在必要时使用 |

#### 2.2.3 角色特质设计（角色卡）

短剧角色需要为**道具、服装、演员**提供清晰指导。角色卡包含：

- **外形设定**：身高/体型/发型/标志性配饰
- **气质标签**：冷面总裁/温柔千金/阴险反派 等
- **说话风格**：语速/口头禅/用词偏好/方言特征
- **情绪表达模式**：愤怒时的微表情/紧张时的小动作
- **服装变化线**：随剧情发展服装颜色/风格的变化（如从朴素到华丽=逆袭线）
- **道具关联**：角色常使用的道具（如总裁的钢笔=权力符号）

## 3. 核心概念：每集=一个分镜场景

### 3.1 设计理念

将每集短剧视为一个**完整的分镜场景单元**。这与传统的"场景→分镜"两步法不同——我们直接在分镜级别生成，跳过中间的"文学剧本"阶段。

**理由**：
1. 短剧每集只有1-3分钟，场景数量有限（通常2-4个），可以直接在分镜级别规划
2. 减少信息损失：从网文→分镜剧本，不需要经过"文学剧本"中间态
3. 更适合AI生成：分镜是结构化的，比纯文学文本更容易约束和评估

### 3.2 每集的结构

```
Episode (1-3 min)
├── 冷开场 (Cold Open, 0-5s)     # 建立冲突/悬念，hook观众
├── 场景A (Scene A, 5-40s)        # 主要场景，可能包含多个镜头
│   ├── Shot 1: 建立镜头 (Establishing)
│   ├── Shot 2-4: 对话/动作镜头
│   └── Shot 5: 反应/情感镜头
├── 场景B (Scene B, 40-70s)       # 冲突升级或新信息引入
│   ├── Shot 1-3: 核心冲突镜头
│   └── Shot 4: Pivot点（情感反转）
├── 尾声 (Tag, 70-90s)            # 收束或留悬念
│   ├── Shot 1-2: 反转展示
│   └── Shot 3: Hook镜头（集尾悬念）
└── 下集预告 (Next Ep Teaser, 可选)
```

### 3.3 每个镜头的信息

每个Shot包含：

| 字段 | 说明 | 示例 |
|------|------|------|
| shot_id | 镜头编号 | EP001-S01 |
| duration | 时长(秒) | 3.5 |
| shot_size | 景别 | close-up |
| camera_move | 运镜 | slow_push |
| angle | 角度 | eye_level |
| subject | 画面主体 | 苏晚晴 |
| action | 动作 | 缓缓抬头，眼神锐利 |
| dialogue | 对白 | "三年了，我终于回来了。" |
| dialogue_style | 对白风格 | 低沉，压抑着愤怒，每个字都咬得很重 |
| narration | 旁白(可选) | "苏氏集团大楼，三年未变。" |
| emotion_tag | 情感标签 | catharsis:7 + oppression:3 |
| sound_design | 音效/配乐提示 | 低沉钢琴渐入，环境音降低 |
| transition | 转场方式 | cut / dissolve / wipe |
| props_in_frame | 画面中的道具 | DNA报告、旧照片 |

## 5. 评估系统设计

### 5.1 评估维度总览

```
评估系统
├── A. 集内评估（单集质量）
│   ├── 爽度值 (Shuang Score)
│   ├── Pivot强度 (情感反转质量)
│   ├── 镜头节奏 (Shot Rhythm)
│   ├── 对白自然度 (Dialogue Naturalness)
│   └── 视觉可行性 (Visual Feasibility)
│
├── B. 集间评估（连续性质量）
│   ├── Hook强度 (Suspense Score)
│   ├── Hook衔接度 (Continuity)
│   ├── 情感弧线连贯性 (Arc Coherence)
│   └── 节奏变化 (Rhythm Variation)
│
├── C. 角色评估（角色一致性）
│   ├── 行为一致性 (Character Consistency)
│   ├── 台词风格一致性 (Speech Consistency)
│   ├── 视觉一致性 (Visual Consistency)
│   └── 角色发展弧度 (Character Development)
│
└── D. 全局评估（整部剧质量）
    ├── 爽度分布 (Shuang Distribution)
    ├── 模式匹配度 (Pattern Fit)
    ├── 付费卡点质量 (Paywall Quality)
    └── 预测留存率 (Retention Prediction)
```

### 5.2 爽度值计算

基于RESEARCH_PLAN.md的公式，适配分镜级别：

```
shuang(t) = |E_release(t) - E_buildup(t-δ)| × trigger_quality(t) × pacing_bonus(t)

其中：
  E_release(t) = 释放维度情感强度 (joy + catharsis + surprise) at shot t
  E_buildup(t-δ) = 蓄压维度情感强度 (oppression + anger + sadness + disgust) at shot t-δ
  trigger_quality = Schulz(2024)的Plot Twist度量：JSD(P(s_{t+1}) ‖ s_{t+1})
  pacing_bonus = 反转间隔在最优范围(3-5个镜头)内时的额外加分(1.0-1.5)
  δ = 蓄压到释放的镜头间隔
```

**短剧特化**：
- 8维情感中的 `oppression（压抑）` 和 `catharsis（爽/解气）` 是短剧独有维度
- 爽度值的计算粒度精确到**镜头级**（而非集级），因为1-3分钟的短剧中情感变化极快

### 5.3 Hook评估

```
Suspense(t) = H(P(s_{t+1}|S_t))

即：观众对下一集第一个镜头（s_{t+1}）的预测分布的熵

高Suspense = 观众完全猜不到下集开头 = 强Hook
低Suspense = 观众能准确预测下集开头 = 弱Hook
```

**集尾Hook质量的3个维度**：

| 维度 | 度量 | 阈值 |
|------|------|------|
| 信息悬念 | 未解决冲突数量 × 信息量 | ≥ 2个未解决冲突 |
| 情感悬念 | 集尾情感状态是否在高位被截断 | 情感强度 ≥ 6/10 时截断 |
| 新悬念引入 | 是否在集尾最后一刻引入新信息 | 至少1个新悬念 |

### 5.4 角色特质评估

用于保证角色在多集之间的一致性：

```
character_consistency = 
  0.3 × speech_style_match    (台词风格是否符合角色卡)
  + 0.3 × behavior_match      (行为是否符合角色性格)
  + 0.2 × visual_match        (视觉表现是否符合角色设定)
  + 0.2 × emotion_match       (情感表达模式是否一致)
```

### 6.2 ScreenplayAgent（分镜生成Agent）详细设计

这是本系统最核心的新增Agent，负责将骨架大纲转化为分镜剧本。

**输入**：
- toc_skeleton.json（目录骨架+每集的情感目标）
- gap_analysis.json（标注了来源：原著/增补）
- character_registry.json（角色档案）
- episode_outline（当前集的大纲）

**生成策略**（基于Wang 2025 TOT + Brahman 2020）：

```
Step 1: 环境生成
  - 根据大纲中的场景描述，生成详细的环境设定
  - 包含：地点/时间/光线/色调/氛围/关键道具
  - 约束：必须与世界观一致

Step 2: 动线设计
  - 根据场景中的角色数量和冲突类型，设计演员走位
  - 规则：
    * 对话场景：两个角色面对面或背对背（根据情感）
    * 冲突场景：角色间距逐渐缩小或扩大
    * 揭露场景：从隐藏到暴露的位置变化
  - 约束：竖屏构图下，角色位置不宜过于复杂

Step 3: 镜头设计
  - 根据情感目标选择景别：
    * 蓄压阶段：中景/全景（展示环境压迫感）
    * 情感爆发：特写/大特写（展示表情细节）
    * 反转时刻：景别突变（全景→特写）
  - 运镜规则：
    * 紧张/悬疑：缓慢推进
    * 动作/冲突：快速摇移
    * 情感/回忆：缓慢拉远
  - 角度规则：
    * 主角正面：平视/微仰（力量感）
    * 反派压迫：俯拍/倾斜
    * 主观感受：主观镜头

Step 4: 对白生成
  - 受情感约束解码（Brahman 2020）：
    * 每句对白的情感分布必须与情感目标一致
    * 角色台词风格必须符合角色卡
  - 短剧对白特征：
    * 短句为主（≤15字/句）
    * 对白节奏快（反应间隔短）
    * 金句设计（可被截图传播的高密度台词）
  - 潜台词标注（给演员的理解提示）

Step 5: 旁白设计
  - 原则：尽量少用旁白，能用画面表达的不用语言
  - 使用场景：
    * 时间跳转（"三年后"）
    * 内心独白（仅在关键情感转折时）
    * 背景信息交代（压缩信息量时）
  - 旁白的语调/风格提示

Step 6: Mock字幕生成
  - 从对白和旁白自动生成SRT格式字幕
  - 包含：时间戳/说话人/文本
  - 时间戳基于镜头时长估算
```

## 10. 设计决策记录

| # | 决策项 | 结论 | 理由 |
|---|--------|------|------|
| D1 | 每集粒度 | 1个分镜场景（2-4个子场景） | 短剧1-3分钟，场景有限，直接分镜级 |
| D2 | 生成方式 | TOT骨架→分镜，不经过文学剧本 | 减少信息损失，更适合AI约束 |
| D3 | 交互模式 | NotebookLM三栏式 | 集成输入/对话/输出，减少上下文切换 |
| D4 | 评估粒度 | 镜头级（非集级） | 1-3分钟内情感变化极快，需细粒度 |
| D5 | Mock字幕 | 自动生成SRT | 为后续配音/字幕团队提供参考 |
| D6 | 角色卡 | 独立生成，含服装变化线 | 服务于道具/服装/演员团队 |
| D7 | 工具箱 | 分步实现，Canvas优先 | Three.js复杂度高，先用2D方案验证 |
| D8 | 对话集成 | 中栏对话直接调用LLM | 实时协作，而非离线生成 |

## 附录C：短剧竖屏构图要点

1. **三分法偏上**：竖屏下主体应偏上1/3，因为手机握持时视觉重心偏上
2. **特写优先**：竖屏天然适合面部特写，充分利用垂直空间
3. **避免宽幅场景**：横幅场景在竖屏中浪费大量上下空间
4. **文字信息放在中下部**：字幕通常在底部，避免与画面信息冲突
5. **双人对话构图**：上下分割而非左右分割（竖屏特有）
